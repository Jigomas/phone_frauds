# Результаты тестирования

## Конфигурация

- ASR: Vosk `vosk-model-small-ru-0.22` (CPU)
- Веса: keyword=0.556, semantic=0.444, prosodic=0.0
- Порог: 0.25
- Выборка для подбора весов: `vishing/samples` (40 файлов, Whisper small)
- Тестовая выборка: `vishing/test` (60 файлов)

## Метрики

| Метрика   | Значение      |
| --------- | ------------- |
| Accuracy  | **68.3%** (41/60) |
| Precision | 65.7%         |
| Recall    | **76.7%**     |
| F1        | 70.8%         |

## Матрица ошибок

|               | Предсказано: Fraud | Предсказано: Legit |
| ------------- | ------------------ | ------------------ |
| **True Fraud** | TP = 23            | FN = 7             |
| **True Legit** | FP = 12            | TN = 18            |

## Наблюдения

- Из 30 мошеннических звонков поймано 23 (76.7% recall)
- 12 легитимных звонков ложно помечены как мошеннические (FP rate = 40%)
- keyword score = 0.00 у большинства файлов — Vosk small пропускает ключевые слова
- Основную нагрузку несёт semantic score (~0.4–0.75 у всех файлов)

## Причина слабой точности

Веса подбирались на `samples/` с ASR **Whisper small**, тест прогнан с **Vosk small** —
разные транскрипции дают другое распределение keyword score.
Порог 0.25 слишком низкий для Vosk: semantic score у легитимных звонков
стабильно попадает в диапазон 0.39–0.56, что превышает порог.

## Что улучшить

1. Перетюнить веса/порог на `vishing/samples` с тем же ASR (Vosk)
2. Расширить словарь триггеров под качество транскрипции Vosk small
3. На машине с GPU запустить с Whisper `large-v3-turbo` — качество транскрипции
   выше, keyword score будет работать лучше
